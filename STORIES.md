# User Stories

## Legende

- **SP** Ein Storypoint entspricht einer Pomodoro Einheit von 40 Minuten.

- **HEAD** Die Verantwortung einer Userstory wird von 
  dieser Person übernommen. Die Tests sollten von einem anderen 
  Teammitglied überprüft werden.

- **Prio** Es muss jeweils mindestens eine *Must Have (MH)*, *Should Have (SH)* und *Nice to have (N2H)* Userstory geben.

- **Status** Der aktuelle Status der User Story basierend auf den GitHub Issues.
